**왜 ONNX로 변환하는가?** Render 무료 서버는 메모리(RAM)가 512MB로 작습니다. PyTorch를 그대로 올리면 무거워서 서버가 죽을 수 있습니다. 학습은 PyTorch로 하고, 실제 서비스(추론)는 가볍고 빠른 ONNX 형식으로 변환해서 올립니다.

```
[Colab: PyTorch로 학습] → [모델을 .onnx 파일로 변환] → [FastAPI: onnxruntime으로 가볍게 추론]
```

## 4단계 — 정상/불량 분류 데이터 만들기

이 데이터셋은 YOLO 형식(이미지마다 불량 위치가 적힌 txt 파일)입니다.
STEP 10에서는 일단 "정상이냐 불량이냐"만 판단하는 모델을 만듭니다 (위치 감지는 STEP 11).

규칙: 라벨 txt 파일이 비어있거나 "good weld" 클래스만 있으면 **정상**, 그 외 클래스가 하나라도 있으면 **불량**.

**셀 4** (경로는 2단계에서 확인한 실제 폴더 경로로 수정하세요):

```python
import os, glob, shutil
from pathlib import Path

# ⚠️ 2단계에서 확인한 실제 경로로 수정하세요
IMG_DIR = 'dataset/train/images'
LBL_DIR = 'dataset/train/labels'

# data.yaml에서 클래스 이름 확인 (있으면)
yaml_files = glob.glob('dataset/**/*.yaml', recursive=True)
if yaml_files:
    print(open(yaml_files[0]).read())

normal_dir = Path('classified/normal')
defect_dir = Path('classified/defect')
normal_dir.mkdir(parents=True, exist_ok=True)
defect_dir.mkdir(parents=True, exist_ok=True)

img_files = glob.glob(f'{IMG_DIR}/*')
count = {'normal': 0, 'defect': 0}

for img_path in img_files:
    stem = Path(img_path).stem
    lbl_path = f'{LBL_DIR}/{stem}.txt'

    is_defect = False
    if os.path.exists(lbl_path):
        with open(lbl_path) as f:
            lines = [l.strip() for l in f.readlines() if l.strip()]
        # 클래스 0번이 good weld가 아니라 다른 클래스가 있으면 불량으로 간주
        # data.yaml 출력 결과를 보고 아래 GOOD_CLASS_ID를 맞게 조정하세요
        GOOD_CLASS_ID = '1'  # 임시값 - data.yaml 확인 후 수정
        for line in lines:
            class_id = line.split()[0]
            if class_id != GOOD_CLASS_ID:
                is_defect = True
                break

    dest = defect_dir if is_defect else normal_dir
    shutil.copy(img_path, dest / Path(img_path).name)
    count['defect' if is_defect else 'normal'] += 1

print(f"정상: {count['normal']}장, 불량: {count['defect']}장")
```

> **중요**: 셀 실행 후 출력된 `data.yaml` 내용에서 클래스 번호를 확인하고, `GOOD_CLASS_ID`를 정확히 맞춰서 다시 실행하세요. 예를 들어 `names: ['bad weld', 'defect', 'good weld']` 라면 `good weld`가 인덱스 2이므로 `GOOD_CLASS_ID = '2'` 로 설정합니다.

## 5단계 — EfficientNet 모델 학습

**셀 5**:

```python
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader, random_split
from torchvision import transforms, models
from PIL import Image
import glob

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print('사용 장치:', device)

# ── 데이터셋 클래스 ──
class WeldDataset(Dataset):
    def __init__(self, root, transform=None):
        self.samples = []
        for label, cls in enumerate(['normal', 'defect']):
            for f in glob.glob(f'{root}/{cls}/*'):
                self.samples.append((f, label))
        self.transform = transform

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):
        path, label = self.samples[idx]
        img = Image.open(path).convert('RGB')
        if self.transform:
            img = self.transform(img)
        return img, label

transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]),
])

full_dataset = WeldDataset('classified', transform=transform)
n_val = int(len(full_dataset) * 0.2)
n_train = len(full_dataset) - n_val
train_ds, val_ds = random_split(full_dataset, [n_train, n_val])

train_loader = DataLoader(train_ds, batch_size=32, shuffle=True)
val_loader   = DataLoader(val_ds, batch_size=32)

print(f'학습 데이터: {n_train}장 / 검증 데이터: {n_val}장')

# ── EfficientNet-B0 모델 (사전학습 가중치 사용 후 미세조정) ──
model = models.efficientnet_b0(weights=models.EfficientNet_B0_Weights.DEFAULT)
model.classifier[1] = nn.Linear(model.classifier[1].in_features, 2)  # 정상/불량 2개 클래스
model = model.to(device)

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)

# ── 학습 루프 ──
EPOCHS = 10
for epoch in range(EPOCHS):
    model.train()
    train_loss, correct, total = 0, 0, 0
    for imgs, labels in train_loader:
        imgs, labels = imgs.to(device), labels.to(device)
        optimizer.zero_grad()
        outputs = model(imgs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        train_loss += loss.item()
        correct += (outputs.argmax(1) == labels).sum().item()
        total += labels.size(0)

    # 검증
    model.eval()
    val_correct, val_total = 0, 0
    with torch.no_grad():
        for imgs, labels in val_loader:
            imgs, labels = imgs.to(device), labels.to(device)
            outputs = model(imgs)
            val_correct += (outputs.argmax(1) == labels).sum().item()
            val_total += labels.size(0)

    print(f'Epoch {epoch+1}/{EPOCHS} | 학습 손실: {train_loss/len(train_loader):.4f} '
          f'| 학습 정확도: {correct/total:.2%} | 검증 정확도: {val_correct/val_total:.2%}')
```

학습이 진행되며 epoch마다 정확도가 출력됩니다. 검증 정확도가 80% 이상이면 합격선입니다.

## 문제 해결

### "다운받은 zip을 GitHub/Colab에 올릴 수 없을 정도로 큼"
Colab 업로드는 보통 문제없지만, 너무 크면(1GB+) Google Drive에 먼저 업로드 후 `drive.mount()`로 불러오는 방법을 안내해 드릴 수 있습니다.

### "학습 정확도는 높은데 검증 정확도가 낮음 (과적합)"
- `EPOCHS`를 줄이거나
- `transforms.RandomHorizontalFlip()`, `RandomRotation(10)` 등 augmentation 추가

### "GPU 안 잡힘 (cuda available: False)"
런타임 → 런타임 유형 변경 → 하드웨어 가속기 GPU 확인. 그래도 안 되면 Colab 무료 GPU 할당량 초과일 수 있으니 잠시 후 재시도.
