# STEP 9 — Vercel(프론트) + Render(AI 서버) 배포 가이드

로컬에서 잘 작동하는 코드를 인터넷에 공개합니다.
- **프론트엔드(Next.js)** → Vercel (무료)
- **AI 서버(FastAPI)** → Render (무료 플랜)

---

## 사전 준비

1. [GitHub](https://github.com) 계정
2. [Vercel](https://vercel.com) 계정 (GitHub으로 가입)
3. [Render](https://render.com) 계정 (GitHub으로 가입)

---

## 1단계 — 코드를 GitHub에 올리기

### 1-1. .gitignore 파일 만들기

프로젝트 루트(`AI 기반 선박 부품 품질검사 웹 플랫폼/`)에 `.gitignore` 파일을 만들고 아래 내용을 붙여넣습니다.

```
# 환경변수 (절대 올리면 안 됨!)
.env
.env.local
.env*.local

# Node
frontend/node_modules/
frontend/.next/

# Python
backend/venv/
backend/__pycache__/
backend/**/__pycache__/
*.pyc
```

### 1-2. Git 초기화 및 커밋

CMD에서 프로젝트 루트 폴더로 이동 후 실행:

```cmd
cd "D:\AI\AI 기반 선박 부품 품질검사 웹 플랫폼"
git init
git add .
git commit -m "첫 커밋: 선박 부품 품질검사 플랫폼"
```

### 1-3. GitHub에 저장소 만들기

1. [github.com/new](https://github.com/new) 에서 새 저장소 생성
   - Repository name: `ship-inspection-platform` (원하는 이름)
   - Private 또는 Public 선택
   - **"Add a README file" 체크하지 말 것**
2. 만들어진 저장소 주소 복사 (예: `https://github.com/yourname/ship-inspection-platform.git`)

### 1-4. GitHub에 푸시

```cmd
git remote add origin https://github.com/yourname/ship-inspection-platform.git
git branch -M main
git push -u origin main
```

## 2단계 — AI 서버를 Render에 배포

### 2-1. Render 웹 서비스 생성

1. [render.com](https://render.com) → **New +** → **Web Service**
2. **Connect a repository** → GitHub 연결 → 저장소 선택
3. 설정 입력:

| 항목 | 값 |
|------|-----|
| Name | `ship-inspection-ai` |
| Root Directory | `backend` |
| Runtime | `Python 3` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `uvicorn main:app --host 0.0.0.0 --port $PORT` |
| Instance Type | `Free` |

4. **Environment Variables** 추가 (아래 버튼 클릭):

| Key | Value |
|-----|-------|
| `DUMMY_MODE` | `true` |
| `FRONTEND_URL` | (Vercel 배포 후 채울 예정, 지금은 비워도 됨) |

5. **Create Web Service** 클릭

> 첫 빌드에 5~10분 걸립니다. 로그에 `Application startup complete.` 뜨면 성공.

### 2-2. AI 서버 URL 확인

배포 완료 후 Render 대시보드 상단에 URL이 표시됩니다.
예: `https://ship-inspection-ai.onrender.com`

브라우저에서 `https://ship-inspection-ai.onrender.com/health` 접속해서
`{"status":"ok"}` 응답이 오면 정상입니다.

> **무료 플랜 주의사항**: 15분간 요청이 없으면 서버가 절전 상태가 됩니다.
> 처음 요청 시 30~60초 웜업 시간이 걸릴 수 있습니다.

## 3단계 — 프론트엔드를 Vercel에 배포

### 3-1. Vercel 프로젝트 생성

1. [vercel.com](https://vercel.com) → **Add New...** → **Project**
2. GitHub 저장소 선택
3. 설정:

| 항목 | 값 |
|------|-----|
| Framework Preset | `Next.js` (자동 감지됨) |
| Root Directory | `frontend` |

4. **Environment Variables** 추가:

| Key | Value |
|-----|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://hbtqhscxexdllsatewyt.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | (Supabase 대시보드에서 복사) |
| `NEXT_PUBLIC_AI_API_URL` | `https://ship-inspection-ai.onrender.com` (Render URL) |

5. **Deploy** 클릭

### 3-2. 배포 완료 후 URL 메모

배포되면 URL이 생깁니다. 예: `https://ship-inspection-platform.vercel.app`

### 3-3. Render에 프론트 URL 등록

Render 대시보드 → ship-inspection-ai → **Environment**:
- `FRONTEND_URL` = `https://ship-inspection-platform.vercel.app`
- **Save Changes** → 자동 재배포됩니다.

## 문제 해결

### Vercel 빌드 실패 시

Vercel 대시보드 → Deployments → 실패한 배포 클릭 → **Build Logs** 확인

흔한 원인:
- 환경변수 이름 오타 → Environment Variables 재확인
- TypeScript 타입 에러 → 로컬에서 `cd frontend && npm run build` 먼저 테스트

### AI 서버에서 CORS 오류 시

브라우저 개발자 도구 Console에 `CORS` 관련 오류가 보이면:
1. Render 환경변수 `FRONTEND_URL` 이 Vercel URL과 정확히 일치하는지 확인
2. Render에서 Manual Deploy → 재배포

### Render 무료 플랜 절전 문제

무료 플랜은 15분 미사용 시 절전합니다. 해결 방법:
- **옵션 A**: 포트폴리오 제출 전날 미리 요청 한 번 날려서 웜업
- **옵션 B**: [UptimeRobot](https://uptimerobot.com) 무료 서비스로 5분마다 핑을 보내서 절전 방지
