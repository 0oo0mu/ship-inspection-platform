## 1. 프로젝트 구조 한눈에 보기

```
사용자 브라우저
    │
    ▼
[Next.js 프론트엔드] ─── Vercel 배포
    │  ├── 로그인 (Supabase Auth)
    │  ├── 선박/블록 등록
    │  ├── 이미지 업로드 → Supabase Storage
    │  └── 검사 결과 조회/대시보드
    │
    ├── (DB 직접) ──────── Supabase PostgreSQL
    │                       선박, 블록, 검사결과 저장
    │
    └── (AI 요청) ──────── [FastAPI 백엔드] ─── Render/Railway 배포
                               │
                               └── AI 모델 (ResNet/YOLO)
                                   이미지 분석 → 정상/불량 판정
```

## 4. 데이터베이스 설계 (Supabase PostgreSQL)

### 테이블 관계도

```
users ──┬── ships ──── blocks ──┬── inspections ──── defect_logs
        │                       │
        └───────────────────────┘
```

### 테이블 상세

#### users
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  role TEXT DEFAULT 'inspector',  -- 'admin', 'inspector'
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### ships (선박)
```sql
CREATE TABLE ships (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,              -- 선박명
  ship_type TEXT,                  -- 선박 유형 (컨테이너, 탱커 등)
  build_number TEXT,               -- 건조 번호
  status TEXT DEFAULT 'building',  -- 'building', 'completed', 'delivered'
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### blocks (블록)
```sql
CREATE TABLE blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ship_id UUID REFERENCES ships(id) ON DELETE CASCADE,
  block_name TEXT NOT NULL,        -- 블록 명칭 (예: A-101)
  process_type TEXT,               -- 공정 유형 (용접, 설치, 조립)
  location_description TEXT,       -- 위치 설명
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### inspections (검사 결과)
```sql
CREATE TABLE inspections (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ship_id UUID REFERENCES ships(id),
  block_id UUID REFERENCES blocks(id),
  user_id UUID REFERENCES users(id),
  image_url TEXT NOT NULL,         -- Supabase Storage URL
  result TEXT NOT NULL,            -- 'normal', 'defect'
  defect_type TEXT,                -- 'crack', 'porosity', 'undercut', 등
  confidence FLOAT,                -- AI 신뢰도 (0.0 ~ 1.0)
  status TEXT DEFAULT 'pending',   -- 'pending', 'rework_requested', 'completed'
  memo TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### defect_logs (불량 위치 - 2단계)
```sql
CREATE TABLE defect_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  inspection_id UUID REFERENCES inspections(id) ON DELETE CASCADE,
  bbox_x FLOAT,                    -- 바운딩 박스 x 좌표 (0~1 비율)
  bbox_y FLOAT,                    -- 바운딩 박스 y 좌표 (0~1 비율)
  bbox_width FLOAT,
  bbox_height FLOAT,
  label TEXT,                      -- 불량 유형
  confidence FLOAT
);
```

## 5. API 설계

### FastAPI 엔드포인트

| 메서드 | URL | 설명 |
|--------|-----|------|
| POST | `/inspect` | 이미지 업로드 → AI 추론 결과 반환 |
| GET | `/health` | 서버 상태 확인 |

#### POST /inspect 요청/응답

**요청 (multipart/form-data):**
```
image: 이미지 파일
```

**응답 (JSON):**
```json
{
  "result": "defect",
  "confidence": 0.92,
  "defect_type": "crack",
  "defect_boxes": [
    {
      "x": 0.3,
      "y": 0.4,
      "width": 0.15,
      "height": 0.1,
      "label": "crack",
      "confidence": 0.92
    }
  ]
}
```

### Next.js → Supabase 직접 호출 (DB/Auth/Storage)

| 동작 | 방법 |
|------|------|
| 로그인/회원가입 | `supabase.auth.signInWithPassword()` |
| 이미지 업로드 | `supabase.storage.from('inspections').upload()` |
| 선박 조회/등록 | `supabase.from('ships').select/insert()` |
| 검사 결과 저장 | `supabase.from('inspections').insert()` |

## 7. AI 데이터셋 구하는 방법

### 옵션 1: AI Hub (추천, 무료)
1. https://aihub.or.kr 접속
2. 회원가입 (기관 이메일 필요 없음, 개인도 가능)
3. 검색: `"용접"` 또는 `"선박 부품"`
4. 추천 데이터셋:
   - **용접부 결함 검출 AI 학습 데이터** — 크랙, 기공, 언더컷 이미지 포함
   - **산업용 표면 결함 검출 데이터** — 금속 표면 불량

### 옵션 2: Kaggle (영어, 무료)
- `Steel Surface Defects` 데이터셋
- `Welding Defect Detection` 검색
- 계정 만들고 바로 다운로드 가능

### 옵션 3: 더미 모델로 먼저 시작 (권장)
실제 데이터 준비 전에 **더미 모델**로 전체 흐름부터 완성한다.
```python
# 더미 AI 응답 예시
def dummy_inspect(image):
    return {
        "result": "defect",  # 항상 불량 반환 (테스트용)
        "confidence": 0.87,
        "defect_type": "crack"
    }
```
→ 나중에 실제 모델로 교체만 하면 됨
